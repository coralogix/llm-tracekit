# Data Model: Strands Agents Instrumentation

**Branch**: `001-strands-agent` | **Date**: 2026-03-09

## Span Entities

### Agent Span (`invoke_agent {agent_name}`)


| Attribute                    | Source                                 | Requirement            |
| ---------------------------- | -------------------------------------- | ---------------------- |
| `gen_ai.system`              | Hardcoded `"strands"`                  | Required               |
| `gen_ai.operation.name`      | `"invoke_agent"`                       | Required               |
| `gen_ai.agent.name`          | `agent.name` from Strands Agents        | Conditionally Required |
| `gen_ai.agent.id`            | Agent instance identifier if available | Conditionally Required |
| `gen_ai.agent.description`   | `agent.description` if available       | Conditionally Required |
| `gen_ai.request.model`       | Model identifier from agent config     | Conditionally Required |
| `gen_ai.agent.tools`         | JSON list of tool names from agent     | Recommended            |
| `gen_ai.usage.input_tokens`  | Aggregated from all cycles             | Recommended            |
| `gen_ai.usage.output_tokens` | Aggregated from all cycles             | Recommended            |


- **Span name**: `invoke_agent {agent_name}` (OTel GenAI semconv: `{gen_ai.operation.name} {gen_ai.agent.name}`)
- **Span kind**: `INTERNAL` (in-process agent orchestration)
- **Parent**: None (root span of trace)

### Cycle Span (`cycle {cycle_id}`)


| Attribute                       | Source                              | Requirement            |
| ------------------------------- | ----------------------------------- | ---------------------- |
| `strands.agent.cycle.id`        | Cycle ID from event loop            | Required               |
| `event_loop.parent_cycle_id`    | Parent cycle ID (recursive/nested)  | Conditionally Required |


- **Span name**: `cycle {cycle_id}` (Strands-specific; no OTel GenAI standard for reasoning cycles)
- **Span kind**: `INTERNAL`
- **Parent**: Agent span (or parent cycle span if nested)

| Attribute                        | Source                          | Requirement              |
| -------------------------------- | ------------------------------- | ------------------------ |
| `gen_ai.system`                  | `"strands"`                     | Required                 |
| `gen_ai.operation.name`          | `"chat"`                        | Required                 |
| `gen_ai.request.model`           | Model name from invocation      | Required                 |
| `gen_ai.response.model`          | Model name from response        | Recommended              |
| `gen_ai.usage.input_tokens`      | From model response             | Recommended              |
| `gen_ai.usage.output_tokens`     | From model response             | Recommended              |
| `gen_ai.usage.cache_read_input_tokens` | From model response (if available) | Recommended        |
| `gen_ai.usage.cache_write_input_tokens` | From model response (if available) | Recommended       |
| `gen_ai.response.finish_reasons` | From model response             | Recommended              |
| `gen_ai.prompt.{n}.role`         | From messages                   | Opt-In (content capture) |
| `gen_ai.prompt.{n}.content`      | From messages                   | Opt-In (content capture) |
| `gen_ai.prompt.{n}.tool_call_id` | Tool result → call correlation  | Opt-In (content capture) |
| `gen_ai.prompt.{n}.tool_calls.{m}.id` | Tool call ID from model response | Opt-In (content capture) |
| `gen_ai.prompt.{n}.tool_calls.{m}.type` | `"function"`                | Opt-In (content capture) |
| `gen_ai.prompt.{n}.tool_calls.{m}.function.name` | Tool function name  | Opt-In (content capture) |
| `gen_ai.prompt.{n}.tool_calls.{m}.function.arguments` | Tool call arguments (JSON) | Opt-In (content capture) |
| `gen_ai.completion.{n}.role`     | From response                   | Opt-In (content capture) |
| `gen_ai.completion.{n}.content`  | From response                   | Opt-In (content capture) |
| `gen_ai.completion.{n}.finish_reason` | From response              | Opt-In (content capture) |
| `gen_ai.completion.{n}.tool_calls.{m}.id` | Tool call ID in completion | Opt-In (content capture) |
| `gen_ai.completion.{n}.tool_calls.{m}.type` | `"function"`           | Opt-In (content capture) |
| `gen_ai.completion.{n}.tool_calls.{m}.function.name` | Tool function name | Opt-In (content capture) |
| `gen_ai.completion.{n}.tool_calls.{m}.function.arguments` | Tool call arguments (JSON) | Opt-In (content capture) |

The `gen_ai.prompt.{n}.tool_calls.*` and `gen_ai.completion.{n}.tool_calls.*` attributes are essential for Coralogix's AI Center to render the "Tool Call" and "Tool Response" conversation segments. They are generated by the core `generate_message_attributes` and `generate_choice_attributes` helpers, consistent with the OpenAI and OpenAI Agents adapters.


- **Span name**: `chat {model}` (OTel GenAI semconv: `{gen_ai.operation.name} {gen_ai.request.model}`)
- **Span kind**: `CLIENT` (external model service call)
- **Parent**: Cycle span

### Tool Span (`execute_tool {tool_name}`)


| Attribute               | Source                              | Requirement              |
| ----------------------- | ----------------------------------- | ------------------------ |
| `gen_ai.system`         | `"strands"`                         | Required                 |
| `gen_ai.operation.name` | `"execute_tool"`                    | Required                 |
| `gen_ai.tool.call.id`   | `toolUseId` from hook event         | Required                 |
| `gen_ai.tool.status`    | Result status (`"success"` / `"error"`) | Recommended          |
| `name`                  | Tool name from invocation           | Required                 |
| `type`                  | Tool type (`"function"` or `"mcp"`) | Required                 |
| `input`                 | Tool call arguments (JSON)          | Opt-In (content capture) |
| `output`                | Tool call result (JSON)             | Opt-In (content capture) |
| `mcp_data`              | MCP metadata (if MCP tool)          | Opt-In (content capture) |


The bare `name`, `type`, `input`, `output`, and `mcp_data` attributes match the OpenAI Agents adapter pattern. No adapter in the codebase uses the newer `gen_ai.tool.name` / `gen_ai.tool.call.arguments` convention attributes. Other adapters (OpenAI, Bedrock, Gemini, LangChain) do not create separate tool spans at all — they record tool definitions as `gen_ai.request.tools.{n}.*` attributes on the LLM call span instead.

- **Span name**: `execute_tool {tool_name}` (OTel GenAI semconv: `{gen_ai.operation.name} {gen_ai.tool.name}`)
- **Span kind**: `INTERNAL` (in-process tool execution)
- **Parent**: Cycle span

## Hook-to-Span Mapping


| Hook Event                             | Span Action                                   |
| -------------------------------------- | --------------------------------------------- |
| `BeforeInvocationEvent`                | Start agent span                              |
| `AfterInvocationEvent`                 | End agent span, record aggregated token usage |
| Cycle start (inferred from event loop) | Start cycle span                              |
| Cycle end (inferred from event loop)   | End cycle span                                |
| `BeforeModelCallEvent`                 | Start model invocation span                   |
| `AfterModelCallEvent`                  | End model invocation span, record attributes  |
| `BeforeToolCallEvent`                  | Start tool span                               |
| `AfterToolCallEvent`                   | End tool span                                 |


## Error Handling

All spans use the core `handle_span_exception(span, error)` pattern, consistent with every other adapter:

| Attribute / Action | Value | When |
| ------------------ | ----- | ---- |
| `span.set_status(ERROR)` | `str(error)` as description | On exception |
| `error.type` | Exception class qualname (e.g. `ModelTimeoutError`) | On exception |
| `span.end()` | — | After recording error |

Error sources from hooks:
- `AfterModelCallEvent.exception` — model invocation failure
- `AfterToolCallEvent.exception` — tool execution failure
- `AfterInvocationEvent` with no `result` — agent-level failure

## Metrics

The adapter emits the standard OTel GenAI metrics via the core `Instruments` class, consistent with all other adapters:

| Metric | Name | Unit | Recorded on |
| ------ | ---- | ---- | ----------- |
| Operation duration | `gen_ai.client.operation.duration` | `s` | Model span end |
| Token usage (input) | `gen_ai.client.token.usage` | `{token}` | Model span end, `gen_ai.token.type = "input"` |
| Token usage (output) | `gen_ai.client.token.usage` | `{token}` | Model span end, `gen_ai.token.type = "output"` |

These feed into Coralogix's AI Center cost tracking and token usage dashboards. The Strands Agents SDK's native metrics (`strands.event_loop.*`, `strands.tool.*`) use custom names that are not recognized by Coralogix's built-in dashboards.

## Relationships

```
Agent Span (1) ──── contains ───→ Cycle Span (1..N)
Cycle Span (1) ──── contains ───→ Model Invocation Span (1)
Cycle Span (1) ──── contains ───→ Tool Span (0..N)
```
